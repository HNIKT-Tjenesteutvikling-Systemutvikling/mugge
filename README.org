#+title: Mugge Chat

* About

This project is a chat application written in Scala 3, using cats-effect for concurrency and fs2 for streaming. It's built with sbt.

* Getting Started

** Prerequisites

- [[https://www.scala-sbt.org/][sbt]] (2.0)
- A JVM

** Building the project

To compile the code, run:
#+begin_src shell
  sbt compile
#+end_src

** Running the client

#+begin_src shell
  sbt client/run
#+end_src

#+begin_src nix-ts
  nix run .#client
#+end_src

* Download a prebuilt client (Linux)

Every =vX.Y.Z= tag publishes a self-contained Linux client to the
[[https://github.com/][GitHub Releases]] page. It bundles its own Java
runtime, so *no JDK or nix is required* on the target machine — any
64-bit glibc Linux works.

#+begin_src shell
  tar xzf mugge-linux-x64.tar.gz
  ./mugge/connect.sh          # connects to the hosted Azure server
#+end_src

- =connect.sh= defaults =CHAT_SERVER_HOST= to the hosted Azure instance;
  override the target with env vars or positional args:
  #+begin_src shell
    CHAT_SERVER_HOST=localhost CHAT_SERVER_PORT=20222 ./mugge/connect.sh
    ./mugge/bin/mugge <host> <port>     # raw launcher, no Azure default
  #+end_src
- *Windows*: run the Linux build under [[https://learn.microsoft.com/windows/wsl/][WSL]].
  There is no native Windows binary.
- *Auth*: this is a team-only client. A provisioned machine
  auto-authenticates via its SSH key + git =user.name=; any machine
  that is not on the allowlist is refused at connect time and
  disconnected — there is no guest access. Ed25519 keys work out of the
  box; RSA OpenSSH keys need =ssh-keygen= (and =openssl= for the PKCS#1
  path) on =PATH=.
- *TLS*: the client speaks TLS 1.3 only and *pins* the server's public
  key (hardcoded in =client/src/chat/Tls.scala=), so a MITM presenting a
  different cert is refused with a clear error. Against a local dev
  server that has no persistent key, pass =--insecure-tls= (or set
  =MUGGE_INSECURE_TLS=1=) to skip pinning — it prints a warning and must
  not be used in production.
- *Voice* needs PipeWire/ALSA on the host; text chat works regardless.
